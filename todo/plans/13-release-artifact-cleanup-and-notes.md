# 方案：发布产物清理与首发说明整理

## 背景

当前已经完成：

- 宿主侧真实写入型 `push --execute` 验证
- 首发版本策略文档
- 发布前检查清单

用户额外提出一个明确要求：

- `todo/` 目录不应该进入最终发布的代码产物

因此本轮要同时处理两件事：

1. 决定并落地发布产物过滤规则
2. 补首个版本的 release note 草案

## 目标

- 让 `todo/` 不进入发布产物
- 补出 `v0.1.0` 的 release note 草案
- 将这两个决定同步到现有发布文档中

## 推荐方案

### 1. 使用发布归档排除规则

推荐新增 `.gitattributes`，至少包含：

- `/todo export-ignore`

这样可以在基于归档的发布产物中排除 `todo/`。

### 2. 保留仓库内管理资料，但不面向发布消费

`todo/` 仍然保留在仓库里，继续用于内部规划与日志。
但发布时应把它视为维护资料，而不是包消费内容。

### 3. 补首发 release note 草案

建议新增单独文档，至少写清楚：

- 首发版本建议：`v0.1.0`
- 已完成的能力范围
- 已完成的验证范围
- 已知限制
- 升级或使用提醒

## 预期结果

- `todo/` 不再属于发布产物
- 首发版本说明可以直接作为 tag / release 的基础文本
- 发布文档之间对首发版本口径保持一致

## 结论

- 状态：执行中
- 当前推荐：新增 `.gitattributes` 与 `v0.1.0` release note 草案
