# 方案目录说明

本目录用于存放需要先讨论、确认后再执行的方案文档。

## 目的

在真正开始代码实现前，把关键方案沉淀为独立文档，便于评审、确认与追踪。

## 使用规则

- 每个需要确认的主题单独建一个文件
- 文件内容使用中文
- 方案未确认前，不进入正式执行阶段
- 方案确认后：
  - 更新 `todo/02-progress.md`
  - 在 `todo/logs/` 记录确认与执行开始时间
  - 再进入实现或迁移工作

## 建议文件命名

- `01-包名与命名空间方案.md`
- `02-Provider-与命令注册方案.md`
- `03-代码迁移边界方案.md`
- `04-本地联调与安装方案.md`
