# 项目总览

## 目标

将当前 BookStack 仓库中的 content sync 能力整理为一个可通过 Composer 安装的外部扩展包，运行在 BookStack 宿主应用内，不要求从 Laravel 依赖中抽离。

## 当前边界

- 宿主：BookStack
- 安装方式：`composer require <package>`
- 运行方式：继续使用 Laravel Artisan 命令
- 不做内容：
  - 不从 Laravel / BookStack 运行环境中抽离核心逻辑
  - 不优先做 Web UI、菜单、权限页面
  - 不尝试合并到 BookStack 主仓库

## 目标形态

### 使用者视角

1. 在 BookStack 项目中安装扩展包
2. 扩展包自动注册 service provider 与命令
3. 用户执行：
   - `php artisan bookstack:pull-content {projectPath}`
   - `php artisan bookstack:push-content {projectPath}`
   - `php artisan bookstack:push-content {projectPath} --execute`

### 实现视角

1. 新建独立仓库维护扩展包
2. 迁移当前 `ContentSync` 相关实现到新包命名空间下
3. 保留对 BookStack / Laravel 宿主环境的依赖
4. 通过 provider 接入容器绑定与命令注册
5. 用本地 path repository 先完成联调，再决定是否发布

## 核心约束

- `todo/` 文档使用中文
- 代码、注释、类名、命令定义使用英文
- 先方案后实现，避免过早迁移代码
- 每次执行都记录到日志文件中

## 风险判断

### 低风险

- 以 Artisan 命令形式接入
- 本地文件读写流程迁移
- 将 `ContentSync` 目录整体迁移到外部包

### 中风险

- 依赖 `BookStack\Http\HttpRequestService` 等宿主内部服务
- BookStack 版本升级导致兼容性变化
- 宿主未来出现同名命令或相近功能

### 暂不处理

- Web 页面入口
- 权限系统扩展
- 完全独立 CLI / PHAR
